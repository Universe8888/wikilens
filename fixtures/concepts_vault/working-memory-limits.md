# Working Memory Limits

Working memory is the mental workspace where conscious thinking happens — where you hold information just long enough to use it. Unlike long-term memory, which can store vast amounts indefinitely, working memory operates under severe capacity constraints.

## Miller's Law: The Magic Number 7 ± 2

In 1956, psychologist George Miller published a landmark paper, *"The Magical Number Seven, Plus or Minus Two"*, in which he observed that humans can reliably hold roughly **5 to 9 discrete items** in working memory at a single moment. Exceeding that ceiling causes earlier items to be displaced before they can be acted upon or transferred to long-term storage.

More recent work by Nelson Cowan (2001) has refined the estimate downward to approximately **four chunks** when rehearsal strategies are controlled, suggesting the effective capacity may be even tighter than Miller proposed.

## Why the Ceiling Matters for Learning

When a learner is exposed to more simultaneous elements than working memory can accommodate, performance degrades noticeably:

- Errors increase as items fall out of the workspace before they can be processed.
- Processing slows because the brain must re-acquire dropped items.
- Long-term retention suffers because inadequately processed material is rarely encoded well.

Instruction that respects these limits — presenting information in smaller, well-sequenced segments — consistently outperforms instruction that front-loads complexity.

## Chunking as the Workaround

The capacity limit applies to *items*, not to the amount of information each item can contain. [[chunking-information]] is the key strategy for working within the constraint: by grouping related pieces into a single meaningful unit, a skilled learner can fit far more raw information into the same number of "slots."

A phone number like `4155552671` occupies ten slots as individual digits; re-encoded as `415 · 555 · 2671` it occupies three — the same constraint, a radically different information density.

## Practical Implications

| Context | Application |
|---|---|
| Classroom instruction | Limit new vocabulary per lesson; introduce concepts one at a time |
| Interface design | Avoid screens with more than ~7 primary action targets |
| Documentation | Break procedures into numbered steps of 5–7 actions |
| Programming | Keep function signatures short; avoid long parameter lists |
